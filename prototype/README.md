# Daily View — 프로토타입

본 구현 시작 전, **UI를 시각적으로 확인**하기 위한 모의 프로토타입입니다.

## 특징

- 더미 항목 10건이 미리 들어있음
- 모든 변경(상태 전이, 코멘트 작성, 새 요청 등록)은 메모리(session_state)상에만 반영
- **브라우저 새로고침 또는 사이드바의 [변경] 후 다시 [저장] 시 초기화**
- 검토자 / 개발자 역할별 화면 차이 확인 가능
- 본 구현 시 이 코드는 버립니다 (참고용)

## 실행 방법

프로젝트 루트(`Daily View/`)에서:

```bash
cd prototype
py -3.12 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

브라우저가 자동으로 `http://localhost:8501`을 엽니다.

> 만약 `py -3.12`가 없다면 `py --version`이 3.12.x인지 확인하고, 그러면 `py -m venv .venv`로 충분합니다.

## 시도해볼 흐름

### 1) 검토자 입장

1. 사이드바에서 이름 `김OO`, 역할 **검토자** → [저장]
2. 대시보드 — 검토 대기, 내가 등록한 미해결, [+ 새 요청 등록] 큰 버튼
3. 좌측 메뉴 **요청목록**에서 카드 그리드, 필터 동작 확인
4. **새요청등록** 메뉴에서 폼 작성, 이미지 첨부 (Win+Shift+S 캡처 → [붙여넣기] 버튼 또는 파일 선택), [등록]
5. **상세보기**는 카드의 [상세보기] 또는 [열기] 버튼으로 진입 (직접 메뉴 클릭 시 ID 없음 안내)
6. 상세 페이지에서 코멘트 입력 → [등록]
7. 상태가 "완료"인 항목에서 **[검토 완료]** 또는 **[재요청]** 버튼

### 2) 개발자 입장

1. 사이드바 [변경] → 이름 `이OO`, 역할 **개발자** → [저장]
2. 대시보드 화면이 자동으로 다르게 — 처리 큐 최상단 (SLA 임박 빨간 띠), [내 큐 보기] CTA
3. 처리 큐의 항목 [열기]
4. 상태 변경 버튼이 검토자와 다름: [→ 확인중], [→ 완료], [→ API대기]
5. 코멘트 작성 시 작성자 이름이 "이OO (개발자)"로 표시됨

### 3) 둘 다 살펴보기

- **요청목록**의 담당자 필터: 개발자로 들어가면 자기 이름이 기본값
- **통계**: 더미 데이터 기반 막대그래프
- **상세보기**의 코멘트 타임라인에 시스템 이벤트(상태 변경)가 점선 박스로 인라인 표시

## 알려진 한계

- 카드/상세 페이지의 이미지는 회색 격자 플레이스홀더로 대체 (실제 디스크 저장 X)
- 새 등록 시 이미지 미리보기는 정상 작동 — 클립보드 붙여넣기, 파일 업로드 둘 다 가능
- 클립보드 붙여넣기는 `localhost` 또는 HTTPS 환경에서만 동작 (브라우저 보안 정책)
- 동시성 처리 X (혼자서만 테스트)
- "통계"는 더미 카운트
- 사용자 식별은 세션 단위 (쿠키 보존 안 함)
