3. 주요 리스크 및 미해결 사항
[A · 치명] HTTPS 미설정 → 사내 다른 PC에서 클립보드 붙여넣기 불가
현상
브라우저 보안 정책상 클립보드 API는 HTTPS 또는 localhost에서만 동작. 현재 RUNBOOK은 평문 HTTP 기반 NSSM 등록 절차만 포함.
영향
검토자가 자기 PC에서 사내 IP로 접속할 경우 핵심 워크플로(캡처→붙여넣기→등록)가 차단됨. 사용자가 "필수 기능"으로 명시한 사항.
대응
mkcert로 사내 root CA 발급 → Streamlit sslCertFile/sslKeyFile 적용 → 각 클라이언트 PC에 root CA 1회 import. 예상 작업량 2~3시간.
[B · 중요] 사내 공용 PC 실배포 미진행
현상
코드·운영 매뉴얼(RUNBOOK.md)은 완비, 그러나 NSSM 서비스 등록·방화벽 8501 인바운드 허용·일일 백업 스케줄러 등록 미수행.
영향
호스트 PC 재부팅 시 서비스 자동 시작 안 됨, 사내 PC에서 접속 경로 미확보, 일일 백업 누락으로 사고 시 복구 불가.
대응
리스크 A의 HTTPS 작업과 일괄 진행 권장. RUNBOOK 5장의 NSSM·방화벽·작업 스케줄러 절차를 순차 적용. 예상 작업량 1시간.
[C · 일반] 실사용자 피드백 미수집
현상
검토자 1명 + 개발자 1명 대상 1주 파일럿 운영이 계획에 있으나 미실시.
영향
실사용 데이터가 없어 M4 우선순위(통계 고도화·스마트 필터 칩·자동 아카이브 강화 등) 판단 근거 부족.
대응
리스크 A·B 해소 후 1주 운영, 사용 빈도·불편 지점·누락 기능을 직접 인터뷰 + 사용 로그 기반으로 수집.
