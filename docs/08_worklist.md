## 작업 명세 (구현 순서)

### 1. 페이지 구조 재편 — `st.navigation` 전환  〔규모 M〕
- **목표 메뉴 구성**
  ```
  대시보드            (app, 메인)
  ───────────────
  새 요청 등록         (빨간 배경 강조)
  ───────────────
  요청목록
  통계
  ```
- 상세보기는 메뉴에서 **숨김** (카드 클릭으로만 진입)
- 영향: `app.py`(진입점에 `st.navigation`+`st.Page`), 각 `pages/*.py`의 `set_page_config` 정리, `st.switch_page` 참조 점검
- 새 요청 등록 강조는 메뉴 아이콘/라벨 + 페이지 내 상단 강조 배너로 보완 (메뉴 항목 배경색은 Streamlit 제약이 있어 CSS 시도)

### 2. 개발자 대시보드 "개발중" 섹션 추가  〔규모 S〕
- 현재: 처리 큐(요청됨·추가확인필요·반려) + 외부 대기(API대기) + 최근 활동
- 추가: **"개발중 (in_progress)"** 섹션 — 개발자가 지금 잡고 있는 작업
- 영향: `app.py` 개발자 분기

### 3. 통계 상태별 카운트 보강  〔규모 M〕
- 현재: 핵심지표가 `진행 중` 하나로 in_progress+api_check+reviewing+needs_recheck+rejected를 전부 합침 → 프로세스별 안 보임
- 추가: 상단에 **상태별 카운트**(요청중/개발중/API대기/검토중/추가확인필요/반려/완료) 한 줄 — 전 단계가 다 잡히게
- 영향: `pages/4_통계.py`

### 4. 담당자 일반화 + 변경 개발자 전용  〔규모 M〕
- "API 담당자" 한정 개념 제거 → 일반 "담당자" (이 건이 지금 누구한테 가 있나를 개발자가 관리)
- `api_check` 진입 시 자동전환 로직 삭제, 프로젝트 설정의 "API 담당자" 입력칸 삭제
- 담당자 **표시**(담당: 홍길동)는 모두에게 유지, **변경(popover)은 개발자 전용**
- 영향: `core/repository.py`(update_status의 api_assignee 분기 제거), `core/project_settings.py`(API담당자 설정 제거), `ui/auth.py`(사이드바 프로젝트 설정의 API담당자 입력 제거), `pages/3_상세보기.py`(담당자 변경 popover를 개발자만)

### 5. DRM 이미지 업로드 실패 안내  〔규모 S〕
- 현재: DRM(문서보안)으로 막힌 이미지 업로드 시 일반 에러
- 변경: 업로드/디코드 실패를 감지해 **"DRM 보호 등으로 이 이미지는 올릴 수 없습니다"** 친절 안내
- 영향: `pages/2_새요청등록.py`, `pages/3_상세보기.py`, (감지 헬퍼는 `core/images.py`)

### 6. 요청목록 카드에 등록 날짜 표시  〔규모 S〕
- 현재: 카드에 "N일 전"(상대시간)만
- 변경: 등록 **절대 날짜**(예: 2026-06-08) 함께 표기
- 영향: `ui/components.py`(render_card)

### 7. 카테고리 기존선택+신규입력 동시 경고  〔규모 S〕
- 현재: 새 요청 등록에서 기존 카테고리 선택 + 신규 입력 동시 시 모호
- 변경: 둘 다 채우면 **"하나만 선택/입력하세요"** 경고
- 영향: `pages/2_새요청등록.py`

### 8. 상세보기 상단 메타를 카드로 묶기  〔규모 S~M〕
- 현재: 상태·등록·긴급도·카테고리 + 변경 버튼이 한 줄 텍스트
- 변경: 이들을 **테두리 카드 한 덩어리**로 묶어 "현재 상태 + 변경" 패널처럼
- 영향: `pages/3_상세보기.py` 상단 메타 영역

### 9. 삭제 버튼 우측상단 이동 + 삭제 목록 필터  〔규모 M〕
- 변경 A: 상세보기 **삭제(보관) 버튼을 우측 상단**으로
- 변경 B: 요청목록에 **"삭제됨만 보기"** 필터 (현재 '삭제 포함' 체크박스를 확장)
- 영향: `pages/3_상세보기.py`, `pages/1_요청목록.py`

### 10. 접속/행동 로그 파일 보강  〔규모 M〕
- 현재: audit 로그에 등록·상태변경·코멘트·이미지 기록 있음. **접속 로그 없음**
- 변경: **접속(페이지 진입/사용자 식별) 로그** 추가, 누락 행동 보강
- 영향: `core/logger.py`(audit), `ui/auth.py`(식별 시점), 각 페이지 진입부
- 화면 없음 — `data/logs/audit.log`에만 적재
