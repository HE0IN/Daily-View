# 03. UI/UX 설계 (Streamlit)

## 3.1 페이지 구성

Streamlit 멀티페이지 자동 라우팅 사용. 파일명 앞 숫자로 사이드바 정렬:

```
app.py                  # 진입점 = 대시보드
pages/
├── 1_요청목록.py
├── 2_새요청등록.py
├── 3_상세보기.py        # ?id=... query param으로 진입
└── 4_통계.py
```

(파일명에 한글이 들어가도 Streamlit 정상 작동. 인코딩은 모두 UTF-8.)

### 사용자 식별 (로그인 없이)
- 사이드바에 이름 텍스트 입력 + 역할 라디오 (검토자 / 개발자)
- "저장" 클릭 시 `st.session_state["user"] = {"name": ..., "role": ...}`
- 브라우저 재방문 시 다시 입력하는 게 번거롭다면, **쿠키 보존** 추가 옵션:
  - `extra-streamlit-components` 의 `CookieManager`
  - 또는 `st.experimental_user` (Streamlit Cloud 한정이라 로컬에선 부적합)
- 이름이 비어 있으면 다른 페이지에서 작업 시도 시 안내 후 사이드바로 유도

```python
# ui/auth.py 핵심
def render_sidebar_user():
    with st.sidebar:
        st.subheader("사용자")
        name = st.text_input("이름", value=st.session_state.get("user", {}).get("name", ""))
        role = st.radio("역할", options=["reviewer", "developer"],
                        format_func=lambda r: "검토자" if r == "reviewer" else "개발자",
                        index=0 if st.session_state.get("user", {}).get("role") != "developer" else 1)
        if st.button("저장"):
            st.session_state["user"] = {"name": name, "role": role}
            st.rerun()
    return st.session_state.get("user")
```

```
[이전 페이지] 1/4 [다음 페이지]
```

### 구현 메모
- `st.columns(3)` 안에서 카드 반복
- 각 카드는 `st.container(border=True)`로 감싸 시각 구분
- 카드 좌측에 긴급도 색상 띠 (HTML+CSS, `st.markdown(unsafe_allow_html=True)`)
- "상세보기" 버튼 클릭 시 `st.query_params["id"] = item.id; st.switch_page("pages/3_상세보기.py")`
- 페이지네이션: 한 페이지 12건 (3열 × 4행)

### 필터 동작
- 필터 위젯 변경 시 자동 rerun됨. 필터 상태는 query param에도 반영하면 새로고침/공유 시 유지.

## 3.6 새 요청 등록 페이지

`st.form` 안에서 작성 → 제출 전까지 rerun 차단으로 입력값 보존.

```
새 요청 등록

[제목 *]                                              [필수]
[__________________________________________]

[설명 *]                                              (마크다운 지원)
┌─────────────────────────────────────────┐
│                                         │
│                                         │
└─────────────────────────────────────────┘

[긴급도 *]    ( )긴급   (●)보통   ( )낮음

[담당 개발자]  [이OO ▼]  (선택)

[태그]        [login, auth, ...] (콤마 구분)

[스크린샷]    [Ctrl+V 붙여넣기]   또는   [파일 선택...]   drag&drop
              미리보기:
              ┌────┐ ┌────┐ ┌────┐
              │    │ │    │ │    │
              └────┘ └────┘ └────┘

[취소]                                          [등록]
```

### 구현 메모
- `st.file_uploader(accept_multiple_files=True, type=["png","jpg","jpeg","webp","gif"])`
- 폼 제출 후 `st.session_state["form_nonce"] += 1`로 위젯 key를 변경해 입력 초기화
  ```python
  nonce = st.session_state.setdefault("form_nonce", 0)
  title = st.text_input("제목", key=f"title_{nonce}")
  ```
- 등록 성공 시 `st.toast("등록되었습니다")` + 목록 페이지로 이동

### 클립보드 붙여넣기 (Ctrl+V) — 검토자 워크플로의 핵심

검토자는 보통 Win+Shift+S로 화면을 캡처한 직후 클립보드에 이미지가 있는 상태에서 등록 폼에 들어옴. 기본 `st.file_uploader`는 클립보드 붙여넣기를 지원하지 않으므로 별도 컴포넌트를 사용한다.

**`streamlit-paste-button`** 사용:

```python
from streamlit_paste_button import paste_image_button as pbutton

paste_result = pbutton(
    label="붙여넣기",
    key="paste_btn",
    background_color="#3B82F6",
    errors="ignore",
)
if paste_result.image_data is not None:
    # PIL.Image.Image 객체 — 그대로 st.image로 표시 가능
    # 디스크 저장 시:
    #   buf = io.BytesIO(); paste_result.image_data.save(buf, format="PNG")
    #   buf.getvalue()를 파일로 기록
    st.image(paste_result.image_data)
```

- 결과는 `PIL.Image.Image` (Pillow 이미지). 본 구현에서 디스크에 저장할 때는 PNG로 직렬화 후 `core/images.py`의 `save_with_thumbnail()` 흐름으로 합류.
- 브라우저 보안 정책상 **localhost 또는 HTTPS** 환경에서만 동작. 사내 IP로 평문 HTTP 접속하면 작동하지 않을 수 있음 → 운영 시 nginx/Caddy 등으로 자체 서명 HTTPS를 두거나 localhost만 권장.
- 등록 폼에서 파일 업로더와 **나란히 노출** → 사용자가 둘 중 편한 방식 선택 (파일은 다중 선택 가능, 붙여넣기는 1회당 1장).
- `pip install streamlit-paste-button` (requirements.txt에 추가).

## 3.9 Streamlit 한계와 회피책

| 문제 | 회피책 |
|---|---|
| 위젯 변경마다 전체 재실행 | 콜백 (`on_change`) 안에서만 무거운 작업 / 외부 I/O |
| `session_state`가 새로고침 시 휘발 | 파일이 단일 진실 소스, session_state는 캐시 |
| 폼 제출 후 입력 초기화가 까다로움 | nonce 카운터를 위젯 `key`에 합성 |
| 파일 업로드가 메모리에 들어옴 | `UploadedFile.getbuffer()` 즉시 디스크 저장, 메모리 보관 X |
| 다른 사용자가 변경한 내용이 반영 안 됨 | `streamlit-autorefresh`로 30~60초 폴링, 또는 수동 새로고침 버튼 |
| 다중 탭/사용자 동시 편집 | 같은 항목 잠금 + 마지막 갱신 타임스탬프 비교, 충돌 시 경고 |
| `st.rerun()` 무한 루프 | 가드 조건 후에만 호출, session_state로 1회성 플래그 사용 |

## 3.10 한글 처리 주의점

- 모든 .py 파일은 **UTF-8 (BOM 없음)** 저장
- 페이지 파일명에 한글 OK, 다만 url에는 percent-encoded로 들어감
- matplotlib 차트가 필요하다면:
  ```python
  import matplotlib
  matplotlib.rcParams["font.family"] = "Malgun Gothic"  # Windows
  matplotlib.rcParams["axes.unicode_minus"] = False
  ```
- 파일 다운로드 (`st.download_button`)에서 한글 파일명은 `urllib.parse.quote()` 처리
- 업로드된 한글 파일명은 슬러그화 후 저장하고 원본은 메타에만 보존
